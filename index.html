Application Guide
Practical guide for using Essam Procurement Solution as a procurement follow-up system.
Beta 5

What this application is for

This app is designed for procurement follow-up: tasks, projects, meetings, notes, contacts, reports, and daily priorities in one place.

The main goal is not to make your team write long reports. The goal is to capture the minimum useful information so you can see what is pending, who is responsible, and what needs action today.

Best rule: every important follow-up should become a task linked to a project, owner, priority, and due date.

What to use it for

Daily follow-upSee today’s urgent work and pending actions.
Project controlKeep each task connected to the right project.
Team assignmentKnow who is handling each procurement item.
Meeting trackingRecord meetings and convert decisions into tasks.
Supplier contactsKeep supplier details and specialties searchable.
ReportsReview workload, status, and overdue actions quickly.

Recommended procurement workflow

1
Create projectCreate the project first so all tasks, meetings, and notes have a clear location.
2
Add taskWrite a clear action title, assign owner, choose priority, set due date, and add attachment/email reference when needed.
3
Follow dailyUse Today’s Focus for tasks and meetings that need attention today.
4
Update statusDo not leave tasks stale. Update status, add comments, or attach the latest reference.
5
Close properlyMark complete only when the required action is actually finished, not just discussed.

Minimum data you should force the team to enter

1Project name
2Task title / required action
3Assigned person
4Due date and priority
5Important reference: file attachment, email link, quotation, or note

Tasks

Tasks are the core of the system. Use them for real actions such as sending RFQ, following quotation, issuing comparison, preparing PO, getting approval, or closing supplier comments.

Avoid vague tasks like “Follow supplier”. Write the action clearly: “Follow ABC for revised furniture quotation for Project X”.

Attachments, email links and Email Task

Use attachment/file link to connect supporting documents such as quotations, comparison sheets, drawings, or approvals.

Use email link/reference when the task depends on an email thread. You can also use Email Task to open a ready email to the assigned team member, and the task will keep a log showing who it was sent to and when.

✓The app prevents repeated identical links inside the same task.
✓Use comments for updates, not for storing every document name.
✓Keep title short, and put details in notes/comments only when needed.

Today’s Focus

This is your execution list for today. It should include today’s due tasks, pinned priorities, and today’s meetings.

Do not put everything in Today’s Focus. If everything is urgent, nothing is urgent.

Hashtags on tasks

Beyond the project, you can tag tasks with hashtags such as #contracts, #payment, or #delivery. This lets you group tasks by type of work across all projects.

Open the Hashtags tab in this guide for the full workflow and recommended tags.

Meetings

Use meetings to record important discussions, coordination points, consultant/client follow-up, supplier meetings, and internal procurement reviews.

After each meeting, convert decisions into tasks. A meeting without actions is only a note, not follow-up.

Recurring meetings

Beta 5 supports repeated meetings such as daily, weekly, or monthly meetings, with an optional repeat-until date.

DDaily: useful for short internal follow-up during critical periods.
WWeekly: useful for project procurement meetings.
MMonthly: useful for management review or supplier performance review.
Do not create recurring meetings for everything. Use it only for meetings that really repeat, otherwise the calendar becomes noise.

Contacts

Use Contacts for suppliers, contractors, service providers, and consultants. Add company, contact person, phone, email, type, specialty, and notes.

The specialties list now supports search and prevents duplicate entries, including duplicates with different letter case or extra spaces.

Specialties

Specialties help you filter suppliers quickly. Keep the list clean and avoid very similar names like “Furniture”, “Furnitures”, and “Loose Furniture” unless you really need separate categories.

Cleaner specialties = faster supplier search.

What are Hashtags?

Hashtags are flexible labels you can attach to any task — across different projects — to group tasks by type of work.

Projects answer the question “which job is this for?”. Hashtags answer “what kind of work is this?”. The two work side by side without conflict.

Example: a task can belong to project Office Renovation and also carry the hashtag #contracts. Another task in Warehouse Setup can also carry #contracts. Now you can see all contract-related work across all projects in one click.

Recommended hashtags for procurement

#contractsAnything tied to a contract that needs review, signature, or follow-up.
#paymentPayment requests, invoices waiting approval, or payment follow-up with finance.
#deliveryPending deliveries, expected shipments, or site delivery coordination.
#rfqRequest for quotation pending from suppliers.
#approvalItems waiting for management or client approval.
#urgent-supplierCritical supplier issues that need immediate attention.
Keep your hashtag list short (5–10 tags). Too many hashtags becomes noise — the same problem as too many specialties.

How to use hashtags

1
Create a hashtagFrom the sidebar, scroll to the Hashtags section and click + New Hashtag. Pick a name and a color.
2
Tag a taskWhen creating or editing a task, click any hashtag chip in the Hashtags field. You can select multiple hashtags per task.
3
FilterClick any hashtag in the sidebar to filter all tasks by that tag. The page title shows the active hashtag, and the count next to each hashtag tells you how many tasks carry it.
4
Edit or deleteHover over any hashtag in the sidebar and click the ⋮ button to rename, change color, or delete it. Deleting a hashtag removes it from all tasks but does not delete the tasks themselves.

Practical use case

You want to know how many contracts are pending across all your projects right now.

1Make sure every contract-related task carries the hashtag #contracts.
2The number next to #contracts in the sidebar is your live count.
3Click #contracts to open the full list, then sort by due date or priority.
4Same workflow works for #payment, #delivery, or any other tag you create.

Hashtags vs Projects — when to use which

Use a Project when…The work belongs to one specific job, location, or contract scope. A task lives in exactly one project.
Use a Hashtag when…You want to group similar types of work across many projects. A task can carry many hashtags.
Rule of thumb: project = the “where”. Hashtag = the “what kind”.

Manager rules

1Do not accept tasks without a project.
2Do not accept “in progress” forever. Every task needs a next action.
3Use overdue tasks as your first review list every morning.
4Use notes for references, but use tasks for actions.
5If the team does not update status, the dashboard will become decoration, not management.

Keyboard / quick use

Use Esc to close this help panel. Use search and filters before creating new duplicate entries.

Blind spot: more features will not fix weak discipline. Keep the process simple and make updates part of the daily routine.
